# 智能图书管理系统（Book System）

基于前后端分离架构的智能图书管理系统，面向图书馆场景实现用户管理、图书管理、借阅管理、分类管理、个性化推荐与封面上传等功能。系统在已有项目基础上完成了迭代优化，重点提升了页面美观度、交互友好性、信息组织能力与推荐结果表达效果。

---

## 一、项目简介

本系统围绕图书借阅业务构建，分为用户端与管理员端两类角色。

- 用户端支持图书检索、查看详情、借阅、归还、查看推荐、维护个人信息等功能
- 管理员端支持图书维护、分类管理、借阅管理、用户管理、封面上传、批量处理等功能
- 系统结合协同过滤、偏好加权、冷启动补偿等方式实现个性化推荐
- 前端在原有基础上进行了页面结构优化与视觉升级，提升了整体可用性与表达效果

---

## 二、项目特色

### 1. 前后端分离
- 前端：Vue 3 + TypeScript + Vite
- 后端：Spring Boot
- 数据库：MySQL

### 2. 双角色设计
- 普通用户：以找书、借书、看推荐为核心
- 管理员：以数据维护、批量处理、资源管理为核心

### 3. 个性化推荐
推荐模块采用混合推荐思路，主要包括：
- 基于借阅行为的协同过滤
- 基于分类偏好的兴趣增强
- 热门图书与新书补充
- 冷启动阶段兜底推荐
- 可选远程模型二次打分
- 推荐理由展示

### 4. 交互优化
- 首页信息总览更清晰
- 图书详情展示更完整
- 推荐结果卡片化展示
- 用户端与管理员端界面区分更明确
- 借阅流程更直观

## 五、推荐逻辑说明

系统推荐逻辑采用“混合推荐”方案，而非单一算法，主要流程如下：

1. 收集用户借阅记录、评分偏好、分类偏好等行为数据
2. 基于用户历史借阅行为构建协同过滤推荐候选集
3. 结合用户偏好分类与兴趣标签做加权增强
4. 在冷启动阶段加入热门图书、新书等兜底候选
5. 如已配置远程推荐模型，则对候选结果进一步评分排序
6. 输出最终推荐列表，并生成推荐理由

这种设计兼顾了：
- 推荐准确性
- 可解释性
- 冷启动稳定性
- 系统可落地性
