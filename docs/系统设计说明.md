# 系统设计说明

## 1. 架构概览
- 前端：Vue 3 + TypeScript + Vue Router + Pinia + Vite
- 后端：Spring Boot + Spring Security + JPA
- 数据库：MySQL
- 文件：图书封面上传到本地目录，由后端静态接口对外提供

## 2. 设计原则
1. 在现有系统基础上渐进升级
2. 先稳业务，再提升体验
3. 优先复用接口，必要时补聚合接口
4. 前端统一视觉与组件规范

## 3. 本轮关键升级
### 3.1 Dashboard 聚合接口
新增 `/api/dashboard/overview`，减少首页多请求拼装，提高首页一致性。

### 3.2 图书详情接口
新增 `/api/books/{id}`，后端直接返回图书详情、可借提示、相似图书，避免前端只靠临时文案兜底。

### 3.3 URL 状态同步
图书页与推荐页对关键状态做 query 同步，支持直接分享与回访。

## 4. 主要模块
- 认证与权限模块
- 用户模块
- 图书模块
- 借阅模块
- 推荐模块
- Dashboard 模块

## 5. 风险控制
- 保持接口兼容
- 高风险操作保留确认
- 前端改造不破坏后端主链路
