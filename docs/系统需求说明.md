# 系统需求说明

## 1. 建设目标
本系统面向“智能图书管理”课题，目标是在现有前后端分离项目基础上，交付一套可稳定运行、可演示、可答辩的 Web 智能图书管理系统。

## 2. 必保核心能力
1. 用户信息管理
2. 图书信息管理
3. 图书检索与借阅
4. 个性化推荐

## 3. 角色划分
### 3.1 普通用户
- 注册、登录、维护个人资料
- 检索图书、查看详情、发起借阅
- 查看借阅状态、续借、归还
- 获取推荐、理解推荐理由

### 3.2 管理员
- 管理图书、封面、分类、用户
- 审核借阅、处理逾期、查看库存
- 查看推荐统计摘要

## 4. 体验目标
- 首页作为真实统一入口
- 用户端与管理端分层明确
- 图书、借阅、推荐三条主线清晰
- 关键页面视觉统一、操作闭环完整

## 5. 非目标
- 不扩展社交、评论、聊天
- 不建设重型 BI 大屏
- 不开发原生移动端 App
